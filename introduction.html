Introduction

Welcome to the Yet Kenya Merchant API documentation. This API enables businesses to facilitate mobile money and card services, including collections and disbursements. With our API, you can seamlessly integrate payment processing capabilities into your applications.

Yet Kenya Limited offers a robust and secure API designed to facilitate mobile money and card services, including collections and disbursements. By integrating with Mpesa and card networks, the API provides businesses with a seamless and efficient way to manage their financial transactions. This integration allows businesses to handle payments, refunds, and other financial operations directly through the Mpesa platform, ensuring fast, reliable, and secure transactions. The API is built to support high-volume transactions, making it ideal for businesses that rely on mobile money as a core part of their operations.

With the Yet Kenya API, businesses can streamline their payment processes, reduce manual errors, and enhance the overall customer experience by leveraging the widespread and trusted Mobile Money Technologies in Kenya.

*Please reach out to Yet Kenya Limited to obtain the baseUrl, username, password, and merchantId.

Definition of Terms

  • baseUrl: The base URL for the API. Will be provided along with credentials.
  • token: A base64-encoded token used for authorization.
  • username: The username provided separately for authentication.
  • password: The password provided separately for authentication.
  • token_clear: A placeholder for a clear token, used in authentication or session management.
  • externalId: This is a unique ID provided by your end to track each individual transaction.
  • callbackUrl: This is a URL from your end that we will call with the result of the payment, whether successful or failed.
  • redirectUrl: This is a URL on your site where the client will be redirected after a successful/failed payment.
  • amount: The amount to be transacted.
  • recipientPhone: The recipient's phone number. The phone number must include the country code.
  • currency: The currency of the transaction.
  • cardNumber: The card number used.
  • expiryMonth: The expiration month of the card.
  • expiryYear: The expiration year of the card.
  • securityCode: The security code of the customer's card.
  • capture: A flag indicating whether to capture the 3DS2 check.
  • merchantId: The ID of the merchant initiating the transfer.
  • payerPhone: The phone number involved in the request.
